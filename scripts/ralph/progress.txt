# Ralph Progress Log

## Codebase Patterns
- Project uses Next.js 15 App Router with TypeScript strict mode
- tRPC v11 with superjson transformer ‚Äî server init in `src/server/trpc/init.ts`, root router in `src/server/trpc/router.ts`
- API route handler at `src/app/api/trpc/[trpc]/route.ts` using fetch adapter
- shadcn/ui with Tailwind CSS v4 ‚Äî use `npx shadcn@latest add <component>` to add components
- Client-side tRPC provider in `src/components/providers.tsx` wrapping React Query
- Path alias `@/*` maps to `./src/*`
- Shared types in `src/types/wastewater.ts` and `src/types/clinical.ts`
- Constants (colors, thresholds, URLs) in `src/lib/constants.ts`
- Drizzle ORM v0.45 with `@vercel/postgres` adapter ‚Äî DB client in `src/server/db/index.ts`, schema in `src/server/db/schema.ts`
- Drizzle uses `drizzle({ client: sql })` pattern (not `drizzle(sql)`) per latest drizzle-orm v0.45
- DB scripts: `db:generate`, `db:migrate`, `db:push`, `db:studio`, `db:seed`
- Clinical indicators use sentinel value `'national'` for department column (not null)
- Sync services in `src/server/services/sync/` ‚Äî accept `db: VercelPgDatabase` parameter, reuse existing fetch functions
- Sync orchestrator route at `src/app/api/sync/route.ts` ‚Äî protected by CRON_SECRET
- Drizzle upserts: use `` sql`excluded.column_name` `` (import `sql` from `drizzle-orm`) in `onConflictDoUpdate.set`
- Drizzle dynamic WHERE: build array of conditions, spread into `and(...conditions)`, pass `undefined` for no filter
- Import `and`, `asc`, `eq`, `gte`, `inArray`, `lte` from `drizzle-orm` (not `drizzle-orm/pg-core`)
- DB columns are snake_case ‚Äî map to camelCase in tRPC responses
- Use `pgTable` third argument callback `(table) => [...]` for indexes/constraints
- `db.insert().values().returning()` returns array ‚Äî destructure `[row]` for single result
- Tests use vitest, co-located with source as `*.test.ts` (e.g., `src/lib/severity.test.ts`) ‚Äî run with `npx vitest run <file>`
- Sickness router at `src/server/trpc/routers/sickness.ts` ‚Äî list/create/delete, all protectedProcedure
- Events router at `src/server/trpc/routers/events.ts` ‚Äî list/create/update/delete, all protectedProcedure, validates category via z.enum
- Recharts already installed ‚Äî used for wastewater chart in `src/components/chart/wastewater-chart.tsx`
- Recharts range bands: use `[lower, upper]` array values in data with a single `<Area>` component (not two stacked Areas)
- Recharts `ReferenceArea`: use `x1`/`x2` with ISO week string values from X axis, `yAxisId` required, `ifOverflow="hidden"` to clip
- Home page is 'use client' at `src/app/page.tsx` ‚Äî uses hooks for state (useDepartmentPreferences, useStationPreferences, etc.)
- Sickness panel at `src/components/sickness/sickness-panel.tsx` ‚Äî shown only when logged in, uses trpc.sickness.list/create/delete
- `usersTable` and `sessionsTable` defined in `src/server/db/schema.ts` ‚Äî sessions FK to users, token is unique
- `bcryptjs` installed for password hashing (with `@types/bcryptjs`)
- `npx` doesn't work for drizzle-kit in this project ‚Äî use `./node_modules/.bin/drizzle-kit` directly
- `npx shadcn@latest` doesn't work ‚Äî use `./node_modules/.bin/shadcn add <component>` directly
- Recharts ReferenceArea/ReferenceLine don't have `hide` prop ‚Äî use `style={{ display: "none" }}` to toggle visibility
- Event chart keys use `event_` prefix + category key (e.g. `event_sick`, `event_gym`) for hiddenKeys toggling
- `React.FormEvent` is deprecated ‚Äî use `React.FormEvent<HTMLFormElement>` instead
- tRPC context uses `cookies()` from `next/headers` ‚Äî passed via `createContext` in `src/server/trpc/init.ts`
- `protectedProcedure` in `init.ts` validates session cookie and adds `ctx.user` ({ id, email })
- Auth router at `src/server/trpc/routers/auth.ts` ‚Äî register/login/logout/me procedures
- Cookie name: `session_token`, HTTP-only, 30-day expiry, lax sameSite
- Event categories constants at `src/lib/event-categories.ts` ‚Äî shared by UI and chart
- EventsPanel component at `src/components/events/events-panel.tsx` ‚Äî replaces SicknessPanel in page.tsx
- shadcn Select component installed via `./node_modules/.bin/shadcn add select`
